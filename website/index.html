What is Toolize?

AI assistants (Claude, Cursor, and any other MCP-compatible client) speak MCP (Model Context Protocol), not OpenAPI. Toolize bridges that gap: import a spec, get ready-to-use MCP tools.

📥

One-click import

Paste a Swagger/OpenAPI URL or upload a YAML/JSON file. Toolize handles the parsing.

⚙️

Automatic generation

Every API endpoint becomes an MCP tool, with its parameters and authentication.

⚡

No code, no restart

Tools are available immediately on /mcp, as soon as the import finishes.

How it works

Four steps between an OpenAPI spec and a tool your AI agent can use.

  1. 1

    Import

    Enter the URL of an OpenAPI/Swagger spec or drop a YAML/JSON file into the Toolize UI.

  2. 2

    Parse

    Toolize analyzes the API's endpoints, schemas, and any authentication mechanisms.

  3. 3

    Expose

    Every endpoint becomes an MCP tool, served as JSON-RPC 2.0 on /mcp.

  4. 4

    Connect

    Point Claude Desktop, Cursor, or any Streamable HTTP-compatible MCP client at that URL.

Quick start

Just one dependency: Docker.

terminal 
docker build -t toolize .
docker run -p 8080:8080 -v toolize-data:/data toolize

Then open your browser at:

url 
http://localhost:8080

Import an OpenAPI/Swagger spec, and the generated MCP tools are immediately available at:

url 
http://localhost:8080/mcp
