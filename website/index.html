concept.md

AI assistants (Claude, Cursor, and any other MCP-compatible client) speak MCP (Model Context Protocol), not OpenAPI. Toolize bridges that gap: import a spec, get ready-to-use MCP tools.

  • 01 · import Paste a Swagger/OpenAPI URL or upload a YAML/JSON file in the web UI. Toolize handles the parsing.
  • 02 · generate Every API endpoint becomes an MCP tool, with its parameters and authentication.
  • 03 · serve Tools are available immediately on /mcp, as soon as the import finishes.

$ curl -X POST localhost:8080/api/projects/import

session.log 
# 1. start
$ docker run -p 8080:8080 -v toolize-data:/data toolize
✓ toolize listening on http://localhost:8080

# 2. import
$ curl -X POST localhost:8080/api/projects/import \
    -H "Content-Type: application/json" \
    -d '{"name":"petstore","openApiUrl":"https://petstore3.swagger.io/api/v3/openapi.json"}'
→ the same import works from the web UI at localhost:8080
✓ parsed 42 endpoints, 12 schemas, 1 auth scheme

# 3. expose
✓ generated 42 MCP tools → served as JSON-RPC 2.0 on /mcp

# 4. connect
$ claude mcp add --transport http toolize http://localhost:8080/mcp
✓ toolize connected
→ point Claude Desktop, Cursor, or any Streamable HTTP-compatible MCP client at that URL

$ docker run toolize
